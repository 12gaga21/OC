# Тестовое окружение для операционной системы

Этот каталог содержит инструменты для локального тестирования операционной системы.

## Структура каталогов

- **unit/** - Юнит-тесты отдельных компонентов ядра
- **integration/** - Интеграционные тесты взаимодействия компонентов
- **performance/** - Тесты производительности
- **scripts/** - Основные тестовые скрипты
- **utils/** - Вспомогательные утилиты для тестирования
- **config/** - Конфигурационные файлы тестов
- **results/** - Результаты выполнения тестов (логи, отчеты)

## Быстрый старт

### Требования
- NASM 2.14 или выше
- QEMU 4.0 или выше
- Bash или совместимая оболочка

### Запуск всех тестов
```bash
cd os/tests/scripts
./run_test.sh all
```

### Запуск конкретных тестов
```bash
./run_test.sh bootloader    # Тестирование загрузчика
./run_test.sh drivers       # Тестирование драйверов
./run_test.sh filesystem    # Тестирование файловой системы
./run_test.sh network       # Тестирование сетевого стека
./run_test.sh shell         # Тестирование оболочки
./run_test.sh performance   # Тесты производительности
```

## Типы тестов

### 1. Юнит-тесты
Тестирование отдельных компонентов ядра:
- Планировщик задач
- Управление памятью
- Драйверы устройств
- Файловая система

### 2. Интеграционные тесты
Тестирование взаимодействия компонентов:
- Загрузка системы
- Работа оболочки
- Сетевые операции
- Межпроцессное взаимодействие

### 3. Тесты производительности
Измерение производительности системы:
- Время загрузки
- Скорость файловых операций
- Производительность драйверов
- Эффективность планировщика

## Конфигурация тестов

Настройки тестов находятся в `config/test_config.yaml`. Основные параметры:

```yaml
qemu:
  path: "qemu-system-i386"
  timeout: 30
  memory: "128M"

tests:
  bootloader: true
  drivers: true
  filesystem: true
  network: true
  shell: true
  performance: false

logging:
  level: "info"
  directory: "../results"
```

## Утилиты тестирования

### `qemu_runner.sh`
Утилита для запуска QEMU с автоматическим сбором логов и обработкой выхода.

### `test_analyzer.py`
Анализ логов тестов и генерация отчетов.

### `test_utils.sh`
Общие функции для тестирования (проверка файлов, сравнение результатов и т.д.)

## Интеграция с CI/CD

Тестовое окружение интегрировано с существующими GitHub Actions workflows:
- `ci-cd.yml` - Основной CI/CD пайплайн
- `performance-test.yml` - Тесты производительности
- `test-drivers`, `test-filesystem`, `test-network` - Специализированные тесты

## Создание новых тестов

1. Создайте файл теста в соответствующем каталоге
2. Добавьте конфигурацию в `config/test_config.yaml`
3. Обновите `run_test.sh` для поддержки нового теста
4. Протестируйте локально
5. Добавьте тест в CI/CD пайплайн

## Пример теста

```bash
#!/bin/bash
# Пример простого теста

source ../utils/test_utils.sh

test_bootloader_size() {
    echo "Тестирование размера загрузчика..."
    
    if [ ! -f "../../os/build/boot.bin" ]; then
        log_error "Файл boot.bin не найден"
        return 1
    fi
    
    size=$(stat -c%s "../../os/build/boot.bin")
    
    if [ $size -eq 512 ]; then
        log_success "Размер загрузчика корректен: $size байт"
        return 0
    else
        log_error "Неверный размер загрузчика: $size байт (ожидается 512)"
        return 1
    fi
}

# Запуск теста
test_bootloader_size
```

## Устранение неполадок

### Ошибка "command not found: qemu-system-i386"
Убедитесь, что QEMU установлен и добавлен в PATH.

### Ошибка "command not found: nasm"
Установите NASM и добавьте в PATH.

### Тесты не запускаются в Windows
Используйте WSL2 или Git Bash для запуска bash-скриптов.

### Нет вывода от QEMU
Проверьте настройки перенаправления последовательного порта в конфигурации теста.

## Лицензия

Тестовое окружение распространяется под той же лицензией, что и основная операционная система.
