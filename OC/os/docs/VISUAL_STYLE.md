# ВИЗУАЛЬНЫЙ СТИЛЬ ПРОЕКТА "UNIVERSAL ASM CORE"

## 🎨 Концепция: "Священный Ритуал"

### Описание стиля
Уникальный визуальный стиль, объединяющий:
1. **Эстетику классических терминалов** (зеленый люминофор на черном фоне)
2. **Имперскую российскую орфографию** (дореформенный русский язык)
3. **Церковную стилистику** (символика †, торжественные формулировки)
4. **Ретро-футуризм** (атмосфера вычислительных машин 1970-80х)

### ⚠️ Юридическая чистота
- ✅ **НЕ использует** торговые марки Warhammer 40k
- ✅ **НЕ копирует** защищенные авторским правом элементы
- ✅ **Вдохновлен** общедоступной эстетикой ретро-терминалов
- ✅ **Оригинальный дизайн** без нарушения прав третьих лиц

## 📋 Технические спецификации

### Цветовая палитра VGA
```nasm
; Основной цвет: 0x0A (Зеленый на черном)
DEFAULT_COLOR_VALUE equ 0x0A

; Особые сообщения: 0x0F (Белый на черном)
WARNING_COLOR_VALUE equ 0x0F

; Предупреждения: 0x0E (Желтый на черном)
ALERT_COLOR_VALUE equ 0x0E

; Критические ошибки: 0x4F (Белый на красном)
ERROR_COLOR_VALUE equ 0x4F
```

### Шрифты и кодировка
- **Кодировка**: CP866 (DOS Russian) с полной поддержкой кириллицы
- **Режим VGA**: 80x25 текстовый режим (BIOS Mode 0x03)
- **Символы**: Поддержка русских букв через таблицу CP866

### Дореформенная орфография
Использовать следующие замены в текстах интерфейса:
- `ый` → `ый` (окончания прилагательных)
- `ий` → `ій` (окончания прилагательных муж.рода)
- `е` → `ѣ` (в отдельных словах: "невѣдомая", "перемѣщеніе")
- `и` → `і` (перед гласными: "інициализированъ")
- `ъ` на конце слов муж.рода: "терминалъ", "процессоръ"

Примеры:
- "Славянский Терминал" → "СЛАВЯНСКІЙ ТЕРМИНАЛЪ"
- "Неизвестная команда" → "Невѣдомая команда"
- "Инициализирован" → "ИНИЦІАЛИЗИРОВАНЪ"

## 🏛️ Стилевые элементы

### Заголовки и приветствия
```nasm
db "† ВСЕМОЩНЫЙ ПРОЦЕССОРЪ ИНИЦІАЛИЗИРОВАНЪ †", 0x0D, 0x0A
db "† СИСТЕМА ГОТОВА КЪ СЛУЖЕНІЮ †", 0x0D, 0x0A
```

### Системные сообщения
Формат: `[КОМПОНЕНТ] Сообщение`
```nasm
db "[СИСТЕМА] Инициализация визуального протокола...", 0x0D, 0x0A
db "[ЯДРО] Синхронизация матрицы дисплея...", 0x0D, 0x0A
db "[РИТУАЛЪ] Терминалъ активенъ. Слава Отечеству!", 0x0D, 0x0A
```

### Приглашение командной строки
```nasm
prompt_string db '[РИТУАЛЬНЫЙ ТЕРМИНАЛЪ]> ', 0
```

### Сообщения об ошибках
```nasm
unknown_command_msg db 'Невѣдомая команда', 0x0A, 0
```

## 🤝 Гайдлайн для ИИ-агентов

### При разработке новых модулей:

1. **Цветовая схема**
   - Использовать `DEFAULT_COLOR_VALUE` (0x0A) для основного текста
   - Применять `ALERT_COLOR_VALUE` (0x0E) для предупреждений
   - Использовать `ERROR_COLOR_VALUE` (0x4F) только для критических ошибок

2. **Тексты интерфейса**
   - Применять дореформенную орфографию во всех пользовательских сообщениях
   - Добавлять символ † для особых/торжественных сообщений
   - Использовать формат `[КОМПОНЕНТ] Сообщение` для системных уведомлений

3. **Названия функций**
   - Сохранять единообразие: `module_action` (например, `vga_init`, `shell_run`)
   - Комментарии писать на русском языке с соблюдением стиля

4. **Пример правильного сообщения**
```nasm
; Правильно:
db "† ЗАГРУЗКА МОДУЛЯ ПАМЯТИ †", 0x0D, 0x0A
db "[ПАМЯТЬ] Выдѣленіе блока... ОК", 0x0D, 0x0A

; Неправильно:
db "Memory module loaded", 0x0D, 0x0A
db "[MEMORY] Allocation complete", 0x0D, 0x0A
```

## 📜 Лицензия стиля

Визуальный стиль "Священный Ритуал" является оригинальной разработкой авторов проекта "UNIVERSAL ASM CORE" и распространяется под той же лицензией, что и весь проект.

Стиль вдохновлен:
- Классическими терминалами 1970-80х годов
- Дореформенной русской орфографией (до 1918 года)
- Эстетикой ранних операционных систем
- Общедоступными элементами ретро-футуризма

**НЕ ЯВЛЯЕТСЯ** производным от произведений Games Workshop,包括但不限于 Warhammer 40,000.
