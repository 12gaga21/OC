# Руководство для ИИ-агента разработки ОС "Universal ASM Core"

## 🎯 Глобальная миссия
Разработать высокопроизводительную операционную систему на чистом ассемблере (NASM), способную работать на **любом устройстве** (от старых ПК до современных серверов) с максимальным **быстродействием**.

## 🔑 Ключевые приоритеты
1.  **Универсальность (Portability):**
    *   Код должен абстрагироваться от железа через **HAL (Hardware Abstraction Layer)**.
    *   Поддержка архитектур: x86 (32-bit), x86_64 (планируется), ARM (планируется).
    *   Поддержка режимов загрузки: Legacy BIOS и **UEFI**.
    *   Автодетект оборудования (ACPI, APIC, PCI/PCIe).

2.  **Быстродействие (Performance):**
    *   Минимизация накладных расходов (zero-overhead abstraction).
    *   Использование **SIMD инструкций** (MMX, SSE, AVX) для вычислений и работы с памятью.
    *   Оптимизированный планировщик задач (Multilevel Feedback Queue).
    *   Асинхронный ввод-вывод (AIO) и кэширование диска.
    *   Поддержка многоядерности (SMP) и локальных переменных на ядро (TLS).

## 🏗️ Архитектурные принципы

### 1. Модульность и Слои
Проект строго разделен на уровни. Нарушение слоев запрещено.
*   **Level 0: Boot & Real Mode** (`src/boot/`) - Загрузчики (BIOS/UEFI).
*   **Level 1: HAL (Hardware Abstraction Layer)** (`src/hal/`) - Прямая работа с железом.
    *   `cpu.asm` (идентификация, MSR, регистры)
    *   `memory.asm` (Paging, PMM, VMM)
    *   `interrupts.asm` (IDT, PIC, APIC)
    *   `drivers/` (Шина PCI, ACPI, таймеры)
*   **Level 2: Kernel Core** (`src/kernel/`) - Логика ОС.
    *   `scheduler.asm`, `vfs.asm`, `ipc.asm`
*   **Level 3: Services & Userland** (`src/user/`) - Оболочка, утилиты, драйверы высокого уровня.

### 2. Стандарты кода (NASM)
*   **Именование:** `snake_case` для функций и меток. Глобальные функции с префиксом модуля (например, `mem_alloc`, `kbd_read`).
*   **Документация:** Каждая функция должна иметь заголовок с описанием входных/выходных параметров (регистры) и используемых регистров (clobbered registers).
*   **Безопасность:** Проверка указателей на NULL, валидация размеров буферов.
*   **Оптимизация:**
    *   Использовать `rep movsq` для копирования памяти вместо циклов.
    *   Выравнивание структур данных по 16 байт для SIMD.
    *   Избегать лишних прерываний в критических секциях.

## 📋 Алгоритм работы агента

При получении задачи агент должен действовать по следующему плану:

### Шаг 1: Анализ контекста
*   Проверить текущую структуру проекта (`tree -L 3`).
*   Убедиться, что задача соответствует глобальной цели (Универсальность/Скорость).
*   Если задача требует изменения архитектуры, сначала предложить план изменений в `DEV_PLAN.md`.

### Шаг 2: Реализация
*   Создавать новые файлы в соответствующих директориях (например, драйверы -> `src/hal/drivers/`).
*   Писать код с соблюдением стандартов именования и документации.
*   Для новых подсистем сразу создавать заголовки-описания (`.md` файлы рядом с кодом).

### Шаг 3: Интеграция
*   Обновить точки входа (например, `kernel.asm` или таблицы вызовов), если добавлен новый модуль.
*   Убедиться, что новый код не ломает существующую сборку (проверить зависимости).

### Шаг 4: Документирование и Коммит
*   Обновить `WORKLOG.md` с описанием сделанного.
*   Сделать коммит с понятным сообщением в формате: `type(scope): description` (например, `feat(hal): add ACPI parser`).
*   **Важно:** После каждого значимого этапа выполнять `git push origin main`.

## 🚀 Текущий фокус (Sprint 1)
1.  **Рефакторинг:** Выделение слоя HAL из текущего ядра.
2.  **Производительность:** Внедрение модуля `simd_ops.asm` для быстрых операций с памятью.
3.  **Совместимость:** Начало работ над поддержкой UEFI (параллельно с BIOS).
4.  **Русский язык:** Полная интеграция и тестирование локализованного вывода.

## ⚠️ Ограничения и запреты
*   **НЕ** использовать Си или другие языки высокого уровня. Только чистый NASM.
*   **НЕ** удалять работающий код без создания резервной копии или ветки.
*   **НЕ** игнорировать обработку ошибок в драйверах.
*   **НЕ** хардкодить адреса памяти (использовать константы из `memory_map.asm`).

## 🛠️ Инструменты сборки
*   Сборка: `nasm -f bin ...`
*   Эмуляция: `qemu-system-i386 -fda build/os.img`
*   Отладка: Использование логов через COM1 (порт 0x3F8) для вывода отладочной информации.
