# ИНСТРУКЦІЯ ПО СОЗДАНІЮ ПРОГРАММЪ

## 1. Введеніе
Священный Компиляторъ «Златой Гласъ» предназначенъ для созиданія бинарныхъ молитвъ (.BIN), которые могут быть исполнены операционной системой «Священный Ритуалъ».

## 2. Требования
- NASM (Netwide Assembler) версии 2.15 или выше
- Bash-оболочка
- ОС «Священный Ритуалъ» или совместимая среда

## 3. Использование

### Базовый синтаксисъ:
```bash
./compiler.sh <файл.asm> [выходной_файл.bin]
```

### Примѣры:

1. **Созиданіе простой программы:**
   ```bash
   ./compiler.sh hello.asm
   # Результат: hello.bin
   ```

2. **С указанным именем выхода:**
   ```bash
   ./compiler.sh myapp.asm sacred_app.bin
   # Результат: sacred_app.bin
   ```

3. **Компиляция тестовой программы:**
   ```bash
   ./compiler.sh hello.asm hello.bin
   ```

## 4. Структура программы на ассемблерѣ

Каждая программа должна имѣть слѣдующую структуру:

```asm
bits 32

start:
    ; Инициализация
    mov ax, 0x18
    mov ds, ax
    
    ; Вызов системных функций через INT 0x80
    mov ax, 0x01      ; Функция вывода
    mov cx, message   ; Строка
    mov dx, msg_len   ; Длина
    int 0x80          ; Вектор Молитвы
    
    ; Завершение
    mov ax, 0x02
    xor bx, bx
    int 0x80
    
    .hang: jmp .hang

message db 'Привѣтствіе!', 0x0D, 0x0A
msg_len equ $ - message
```

## 5. Системные вызовы (Векторъ Молитвы INT 0x80)

| AX | Функция | Описание | Входные параметры |
|----|---------|----------|-------------------|
| 0x01 | sys_print | Вывод строки | BX=сегмент, CX=смещение, DX=длина |
| 0x02 | sys_exit | Завершение процесса | BX=код возврата |
| 0x03 | sys_read | Чтеніе ввода | BX=буфер, CX=длина |
| 0x04 | sys_write | Запись в файл | BX=дескриптор, CX=буфер, DX=длина |
| 0x05 | sys_open | Открытіе файла | BX=имя, CX=флаги |
| 0x06 | sys_close | Закрытіе файла | BX=дескриптор |

## 6. Священная Библіотека (SYSLIB)

Для упрощенія программированія рекомендуется использовать SYSLIB:

```asm
%include "syslib.asm"

start:
    call print_string    ; Макрос SYSLIB
    call exit            ; Макрос SYSLIB
```

## 7. Инквизиція Кода

Перед компиляціей проводится проверка:
- ✅ Целостность исходнаго файла
- ✅ Наличие точки входа `start`
- ✅ Корректность синтаксиса NASM
- ⚠️ Проверка на «еретическіе» конструкціи (будущее расширеніе)

## 8. Устраненіе Неисправностей

**Ошибка: «Исходный файл не обрѣтенъ»**
- Проверьте путь к файлу
- Убѣдитесь, что файл имеет расширеніе `.asm`

**Ошибка: «Неудача при сборкѣ»**
- Проверьте синтаксис NASM
- Убѣдитесь, что NASM установлен (`nasm --version`)
- Проверьте наличие включаемых файлов (%include)

**Предупрежденіе: «Священная Библіотека не найдена»**
- Программа будет скомпилирована без линковки SYSLIB
- Используйте полные адреса системных вызовов
