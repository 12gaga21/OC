# Архитектура ядра операционной системы

## Общая архитектура

Ядро операционной системы полностью написано на ассемблере NASM и следует модульной архитектуре, где каждый компонент реализован в отдельном файле. Ядро обеспечивает базовые функции операционной системы, включая управление памятью, многозадачность, обработку прерываний и системные вызовы.

## Структура ядра

### Основные компоненты

1. **Точка входа (kernel.asm)** - основная функция ядра и обработчики прерываний
2. **Вспомогательные функции (kernel_utils.asm)** - функции работы со строками и памятью
3. **Функции работы с числами (number_utils.asm)** - преобразование чисел в строки и обратно
4. **Информация о системе (system_info.asm)** - получение информации о CPU и памяти
5. **GDT (gdt.asm)** - реализация Global Descriptor Table
6. **IDT (idt.asm)** - реализация Interrupt Descriptor Table
7. **Многозадачность (task.asm)** - система управления задачами
8. **Менеджер памяти (memory.asm)** - система управления памятью
9. **Системные вызовы (syscall.asm)** - обработчики системных вызовов

### Архитектура памяти

Ядро использует плоскую модель памяти в 32-битном защищенном режиме. Стек ядра расположен по адресу 0x90000 и имеет размер 16KB. Куча ядра начинается с адреса 0x100000.

### Система задач

Ядро поддерживает до 16 задач, каждая из которых имеет собственный стек и состояние. Переключение задач происходит в основном цикле ядра с использованием round-robin алгоритма.

### Система управления памятью

Менеджер памяти реализует простую систему выделения и освобождения блоков памяти. Память выделяется из кучи ядра, которая начинается с адреса 0x100000.

### Система прерываний

Ядро использует IDT для обработки прерываний и исключений. Реализованы обработчики для всех 32 исключений процессора. Также реализованы обработчики для аппаратных прерываний от клавиатуры и таймера.

### Системные вызовы

Ядро предоставляет интерфейс системных вызовов для взаимодействия с приложениями. Реализованы вызовы для печати строк, создания задач и завершения задач.

## Интерфейсы компонентов

### Вспомогательные функции

- `strlen` - вычисление длины строки
- `strcpy` - копирование строки
- `strcmp` - сравнение строк
- `memset` - заполнение памяти значением
- `memcpy` - копирование памяти

### Функции работы с числами

- `int_to_string` - преобразование целого числа в строку
- `string_to_int` - преобразование строки в целое число
- `hex_to_string` - преобразование числа в шестнадцатеричную строку

### Информация о системе

- `get_cpu_info` - получение информации о процессоре
- `get_memory_info` - получение информации о памяти
- `print_system_info` - вывод информации о системе

### Многозадачность

- `task_init` - инициализация системы задач
- `task_create` - создание новой задачи
- `task_switch` - переключение задач
- `task_get_current_id` - получение ID текущей задачи
- `task_set_state` - установка состояния задачи

### Менеджер памяти

- `memory_init` - инициализация менеджера памяти
- `malloc` - выделение памяти
- `free` - освобождение памяти
- `memory_info` - вывод информации о памяти

### Системные вызовы

- `syscall_handler` - обработчик системных вызовов
- `syscall_init` - инициализация системы системных вызовов

## Поток управления

1. Загрузчик загружает ядро и передает ему управление
2. Ядро инициализирует GDT и IDT
3. Ядро инициализирует систему задач и менеджер памяти
4. Ядро инициализирует систему системных вызовов
5. Ядро инициализирует драйверы устройств
6. Ядро инициализирует файловую систему и сетевые возможности
7. Ядро инициализирует оболочку
8. Ядро включает прерывания
9. Ядро входит в основной цикл, где запускает оболочку и переключает задачи

## Расширяемость

Архитектура ядра позволяет легко добавлять новые компоненты и функции. Для добавления нового компонента достаточно создать новый файл с реализацией и добавить его в систему сборки.
